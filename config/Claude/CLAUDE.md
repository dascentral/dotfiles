# CLAUDE.md

## Tone and Tact

- Be positive but objective and constructively critical.
- Do not give unearned agreement or praise.

## Punctuation

- Prefer clear punctuation using periods, commas, and parentheses. Avoid em dashes unless needed for clarity.

## Tooling

- Prefer platform native tools for GitHub workflows, for example `gh`, when available.

## Environment Specific Overrides

Use this section only when the current environment supports these tools and workflows.
If a tool is unavailable, fall back to the closest supported alternative and continue.

### Cursor IDE Environment

1. Prefer Cursor native tools for reading and editing files.
2. For GitHub workflows, use `gh` when available.
3. For browser automation, prefer the configured browser agent workflow for this environment.
